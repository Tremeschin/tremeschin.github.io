Generally speaking:

- Coding bandwidth is not a bottleneck, but rather worthy ideas and maintenance.
- Documentation is always starving for help, my focus is on the developer experience.
- Good issues or a quick message are desirable over pull requests for me to implement[^pull]. Some repositories are under contributors-only mode when on internal changes or plans.

[^pull]: Fixes are trivial for ones with experience and codebase knowledge; Minor things like typos, version bumps does not matter; Features always need thought on scope, feasibility, maintenance, before implementation.
